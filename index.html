今さらのUnite.vim

っていうかVimって？

Vim（ヴィムまたはヴィアイエム）は、vi から派生し、発展した高機能なテキストエディタである。
VimはGUIを必要とせずCUIでも動くため、Unix系OSに標準のエディタとして搭載されていることが多い。
viエディタと同様、キーボードのみで操作されることを前提としていたため、キーボードのみですべての操作が可能になっている。
(Wikipediaより)

つまり SO COOL!!!

  • 高いテキスト編集能力
  • やっぱりカーソル移動って言ったらhjklだよねー
  • マウス(笑)
  • プラグインによる拡張

Unite.vimって？

  • こんなこというひとがつくった超有名Vimﾌﾟﾗｷﾞﾝ

  • vim 用の統合ユーザインターフェース。様々なデータを共通のインタフェースを用いて操作できる。
  • source(file等)を選択後、候補を絞り込み、actionを選択する。
    • :Unite <source>
  • :Unite file bufferとか打ち込むと何ができるわかるかも。
  • 自分で拡張できる。
  • とはいえ意味わかんないですよね。

多分わかりやすい例

supermomonga/jazzradio.vim

VimでJAZZRADIO.COMにある曲を聞こうというマジ○チプラグイン。
mplayerに依存しているので試したい方は 

$sudo apt-get install -y mplayer  
$brew install mplayer

などのあと、.vimrcに

NeoBundle 'supermomonga/jazzradio.vim'

など追記してください

  • インストールするとsourceにjazzradioが追加される。
  • :Unite jazzradioと打ち込むとチャンネル一覧が表示される。
  • 今日はボサノヴァの気分♪ ってときには適当に"bossa"とか打ち込むと該当するチャンネルが絞り込まれる。
  • 前述したactionに何も選択しないとsourceデフォルトのaction(jazzradioの場合はplayというsource独自のaction)が選択される。
  • 目標(好きなチャンネル)をセンター(カーソル)に入れてスイッチ(エンター)
  • 選択したチャンネルで放送中の曲が聞ける！！

つまり

  • :Unite <source>で候補リストを表示
    • sourceは候補リストを作成
  • 絞り込み(sourceによっては複数選択可)
  • actionを選択
    • 独自拡張にはsourceに紐付いた専用actionが用意されていることが多いが、echoなど汎用actionも

sourceの役割 ≒ 候補リスト作成 & action定義

→ 定義さえすれば同一インターフェイスで様々なことができる
